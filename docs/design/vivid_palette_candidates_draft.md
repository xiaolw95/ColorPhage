# ColorPhage `vivid` 首批候选 Palette 草案

## 1. 文档目的

本文件用于整理 ColorPhage 简单模式 `vivid` 的首批候选方向。

当前阶段先从 `vivid_core` 开始，目标不是做出“更鲜艳的 `soft`”，而是建立一种：

- 有明确视觉锚点
- 冷暖对比更强
- 跨色系但仍有秩序
- 适合科研主结果图

的受控高对比 palette。

## 2. `vivid` 当前设计原则

`vivid` 模式当前遵循以下原则：

- 允许更大的色相跨度
- 必须存在 1 到 2 个清晰视觉锚点
- 可以使用中高饱和度颜色，但不能变成海报风或彩色糖豆风
- 整体仍然要像科研图，而不是商业品牌图

## 3. `vivid_core` 初版方向

### 3.1 定位

`vivid_core` 是 `vivid` 模式当前最适合做默认入口的一支。

它的目标不是“最戏剧化”，而是：

- 在科研图里足够醒目
- 在对照组与处理组、多亚型、多分组图里能迅速拉开差异
- 仍然维持较高的完成度和秩序感

### 3.2 当前候选颜色

```text
#1D3557
#457B9D
#A8DADC
#F1FAEE
#E63946
```

### 3.3 当前气质

- 深海军蓝作为压轴主色
- 中蓝与浅青构成冷色骨架
- 极浅冷白提供缓冲
- 珊瑚红作为最强视觉锚点

### 3.4 主要参考来源

- `Color vol.02｜高级感配色_1_是韩颖啊_来自小红书网页版.jpg`
- `0 (8).png`
- `0 (4).png`

### 3.5 适用场景

- 处理组 vs 对照组
- 多亚型主结果图
- 强调主结论的折线图、散点图、柱状图

### 3.6 当前风险

- 红色锚点过强时，可能压过其他组
- 浅冷白在真实图中更适合作为弱化组，而不适合作为需要强识别的主组
- 若扩展到高类别数，必须单独设计扩展逻辑

## 5. 当前下一步建议

1. 先生成 `vivid_core` 的网页预览
2. 检查整体是否符合你对 `vivid` 的直觉
3. 再决定是否继续做 `vivid_fresh` 或 `vivid_drama`

## 6. 当前阶段结论

截至当前阶段：

- `vivid_core` 已被用户判断为“基本符合，可以继续”
- `vivid_core` 更像用户想要的默认 `vivid`

因此当前建议：

- 保留 `vivid_core` 作为 `vivid` 模式的默认优先候选
- 保留 `vivid_fresh` 作为清爽分支候选
- 后续是否继续设计 `vivid_drama`，可根据体系完整性再决定

## 7. `vivid_fresh` 初版方向

### 7.1 定位

`vivid_fresh` 是 `vivid` 的清爽支线。

它的目标是：

- 保留 `vivid` 的清晰对比
- 比 `vivid_core` 更轻、更明快
- 更适合生物、环境、组学、生态相关图

### 7.2 当前候选颜色

```text
#0F6D7A
#83C5BE
#EDF6F9
#FFDDD2
#E29578
```

### 7.3 当前气质

- 深青蓝作为稳定主轴
- 浅青绿提供生命力
- 冷白降低压迫感
- 浅桃与暖陶色负责温暖对比

### 7.4 主要参考来源

- `Color vol.02｜高级感配色_2_是韩颖啊_来自小红书网页版.jpg`
- `0 (2).png`
- `0 (5).png`

### 7.5 与 `vivid_core` 的区别

- `vivid_core` 更像主结果图的对照强化
- `vivid_fresh` 更像清爽、开放、偏生命科学语境的明快强化

### 7.6 当前风险

- 若控制不好，可能过于温柔而失去 `vivid` 的力度
- 浅桃与暖陶色之间需要关注层级是否足够

## 8. 当前排序建议

当前 `vivid` 模式内部排序建议为：

1. `vivid_core`
2. `vivid_fresh`

当前判断依据：

- `vivid_core` 更接近用户期待的默认 `vivid`
- `vivid_core` 更适合作为简单模式中“强烈”路径的标准入口
- `vivid_fresh` 保留了高完成度和清爽感，更适合作为风格分支而非默认

## 9. `vivid_drama` 重做方向

### 9.1 定位

`vivid_drama` 是 `vivid` 的戏剧支线，但在这一轮重做之后，它不再追求“更亮、更炸”的戏剧性，而是转向：

它的目标是：

- 比 `vivid_core` 更有类别张力
- 比旧版 `vivid_drama` 更低亮度、更低饱和度
- 更接近高水平学术期刊中多类别分群图、UMAP 图、系统发育分类图的“成熟强对比”
- 保留科研图边界，而不是滑向海报风

### 9.2 本轮用户反馈

用户对上一轮 `vivid_drama_rework` 的反馈非常明确：

- 色系方向可以接受
- 但整体亮度和饱和度都偏高
- 不符合“高级学术期刊”的成熟科研审美

因此这一轮重做的核心，不是继续增强冲击力，而是：

- 压低明度
- 压低纯度
- 保留足够强的色相分离
- 让 palette 在大面积使用时也不显得浮夸

### 9.3 当前候选颜色

```text
#4B6A97
#5B9A90
#8FAF62
#D3A35A
#C97B5D
#8C6492
```

### 9.4 当前气质

- 烟蓝作为稳定冷色锚点
- 灰青和苔绿构成中段过渡层
- 老金和陶土色负责暖色区分
- 烟紫作为末端高区分补位
- 整体更像“压过高亮之后的成熟多类别 palette”

### 9.5 主要参考来源

- `Color vol.02｜高级感配色_1_是韩颖啊_来自小红书网页版.jpg`
- `微信截图_20241120123537.jpg`
- `微信截图_20231014103647.png`
- `微信截图_20231014103659.png`
- `support_20231014103724.png`

### 9.6 与前两支的区别

- `vivid_core` 更稳、更通用
- `vivid_fresh` 更轻、更透气
- `vivid_drama` 更适合多类别、高区分场景，但不再依赖高纯度颜色取胜

### 9.7 当前风险

- 若类别数继续上升到 8 以上，仍需单独做扩展系统
- 苔绿与老金在某些缩小场景下需要关注相邻分组的距离
- 它更适合多类别点云、分群、系统发育图，不一定适合作为所有柱状图默认强烈方案
