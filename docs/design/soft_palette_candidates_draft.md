# ColorPhage `soft` 首批候选 Palette 草案

## 1. 文档目的

本文件用于整理 ColorPhage 简单模式 `soft` 的首批候选 palette。

当前目标不是立即定稿，而是基于已经完成初筛的高价值参考图，先提出 3 组方向明确、可继续测试的候选组合。

## 2. 设计原则

`soft` 模式当前遵循以下原则：

- 柔和、精致、克制、耐看
- 以低到中低饱和度为主
- 明度以中高为主
- 冷暖可混合，但整体语气必须统一
- 颜色之间要有区分，但不能过分跳跃
- 适合论文主文统计图、分组图、长期观看场景

## 2.1 关于颜色数量扩展

当前文档中先给出 `n = 3`、`n = 4`、`n = 5` 的建议顺序，是为了优先建立第一阶段最常见场景下的可用方案。

但 `soft` 模式后续不能只服务于低类别数。

需要明确支持的长期方向包括：

- `n = 6`
- `n = 7`
- `n = 8`
- `n = 10`
- `n = 12`
- 以及更多类别的扩展场景

例如：

- 菌群分类图
- 单细胞聚类注释图
- 多处理组合图
- 多亚型分组图

因此：

- 当前 5 色方案应视为“种子 palette”
- 后续需要单独设计高类别数扩展版本
- 不能简单依赖插值补色
- 高类别数应采用更强调秩序和分层的策略

## 3. 当前候选草案

### 3.1 `soft_breeze`

定位：

- 柔和青绿主导的平衡型 `soft`
- 适合作为简单模式的默认候选之一

候选颜色：

```text
#0C9A89
#E796A1
#F6D8E5
#C7E1F4
#F5CB7E
```

主要气质：

- 温柔
- 清新
- 有轻微生命科学气息
- 适合跨色系但不激进的科研图

主要参考来源：

- `0 (1).png`

可借鉴逻辑：

- 用青绿作为稳定主色
- 粉与浅粉作为柔和补充
- 浅蓝和柔黄提供层次与轻度对比

适合场景：

- 分组柱状图
- 分类散点图
- 轻量堆叠图

潜在风险：

- 如果类别很多，粉色与浅粉可能区分度下降
- 需要测试在浅底图中的对比是否足够

当前建议取色顺序：

原始候选颜色：

```text
1. #0C9A89
2. #E796A1
3. #F6D8E5
4. #C7E1F4
5. #F5CB7E
```

#### `n = 3`

建议顺序：

```text
#0C9A89
#E796A1
#C7E1F4
```

说明：

- 青绿作为稳定主色
- 豆沙粉提供温和暖色锚点
- 浅蓝作为冷色补充，保持整体轻盈

#### `n = 4`

建议顺序：

```text
#0C9A89
#E796A1
#C7E1F4
#F5CB7E
```

说明：

- 在 `n = 3` 的基础上补入柔黄
- 让 palette 在保持轻柔的同时增加一点活力
- 比直接加入浅粉更有利于拉开组间差异

#### `n = 5`

建议顺序：

```text
#0C9A89
#E796A1
#F6D8E5
#C7E1F4
#F5CB7E
```

说明：

- 保留完整 palette
- 浅粉作为最柔和的一位，更适合放在弱化组或辅助组
- 若图中 5 组都需要强识别，需谨慎使用浅粉

当前使用建议：

- `n = 3` 和 `n = 4` 是 `soft_breeze` 最稳的用法
- `n = 5` 时建议结合图例排序和数据语义安排浅粉位置
- 若后续发现浅粉在真实图中偏弱，可考虑替换为更清晰的浅紫粉或灰粉

### 3.2 `soft_harbor`

定位：

- 雾蓝灰与暖橙粉平衡的成熟型 `soft`
- 更偏“科研感 + 高级感”的稳妥方案

候选颜色：

```text
#92ADC0
#9DBCBA
#F1B04A
#EB6E51
#F3B1A6
```

主要气质：

- 稳定
- 克制
- 有轻微叙事感
- 比 `soft_breeze` 更成熟、更像论文图

主要参考来源：

- `0 (3).png`

可借鉴逻辑：

- 以前两位冷色作为秩序骨架
- 用柔黄和珊瑚橙作为暖色层级
- 最后以浅粉收尾，保持柔和而不生硬

适合场景：

- 多组折线图
- 箱线图
- 方法比较图

潜在风险：

- 黄橙相邻时可能需要依赖顺序安排保证区分
- 若全部用于面积填充，暖色段可能显得稍重

### 3.3 `soft_mist`

定位：

- 淡紫、雾蓝、米白、浅粉构成的低冲突 `mist` 倾向 `soft`
- 更偏安静、轻盈、长期观看友好

候选颜色：

```text
#C5B4D8
#CDDDE5
#EEE9E0
#F4C1BF
#EAD2D2
```

主要气质：

- 安静
- 雾感
- 轻盈
- 更接近 `mist` family 的延伸

主要参考来源：

- `0 (6).png`

可借鉴逻辑：

- 用淡紫与雾蓝构成冷静基础
- 米白降低冲突感
- 两个粉色负责温度与柔和层次

适合场景：

- 辅助图
- 主文中的温和比较图
- 少组别点图或线图

潜在风险：

- 整体过于接近，可能在 5 组同时出现时区分度不够
- 更适合作为 `n = 3` 或 `n = 4` 的 `soft` 候选

当前建议取色顺序：

原始候选颜色：

```text
1. #C5B4D8
2. #CDDDE5
3. #EEE9E0
4. #F4C1BF
5. #EAD2D2
```

#### `n = 3`

建议顺序：

```text
#C5B4D8
#CDDDE5
#F4C1BF
```

说明：

- 以淡紫和雾蓝构成冷静基础
- 用浅粉提供柔和暖色锚点
- 避免在 3 色方案中加入过于接近的米白或浅粉尾色

#### `n = 4`

建议顺序：

```text
#C5B4D8
#CDDDE5
#F4C1BF
#EEE9E0
```

说明：

- 在前三色基础上补入米白
- 让第四组作为最弱、最背景化的一组更自然
- 更适合图中本身存在主次层级时使用

#### `n = 5`

建议顺序：

```text
#C5B4D8
#CDDDE5
#EEE9E0
#F4C1BF
#EAD2D2
```

说明：

- 保留完整 palette
- 该方案主要用于需要非常柔和气质的图
- 如果五组都要求强识别，不建议优先使用这一组

当前使用建议：

- `soft_mist` 最适合 `n = 3` 或 `n = 4`
- `n = 5` 时更适合作为补充图、辅助图或语义主次分明的图
- 若未来保留为正式 palette，更适合归入 `mist` 倾向分支

## 4. 当前初步判断

从当前 3 组草案来看：

- `soft_breeze` 最有“清新型入口”的潜力
- `soft_harbor` 最有“成熟科研图”质感
- `soft_mist` 最有 `mist` 倾向，但需要更谨慎处理区分度

## 5. 当前默认优先候选

经当前阶段讨论，暂定：

- `soft_harbor` 为 `soft` 模式的默认优先候选

当前排序建议：

1. `soft_harbor`
2. `soft_breeze`
3. `soft_mist`

当前判断依据：

- `soft_harbor` 的冷暖平衡最稳定
- `soft_harbor` 更像适用于广泛科研场景的默认科研图配色
- `soft_harbor` 相比 `soft_breeze` 更稳重，容错率更高
- `soft_mist` 更适合作为偏 `mist` 的分支，而不是默认方案

## 6. `soft_harbor` 当前建议取色顺序

为兼顾区分度和整体平衡，当前建议 `soft_harbor` 的默认取色顺序不是严格按原始列顺序使用，而是按“先骨架、再补层次、最后补柔和色”的思路取色。

原始候选颜色：

```text
1. #92ADC0
2. #9DBCBA
3. #F1B04A
4. #EB6E51
5. #F3B1A6
```

### 6.1 `n = 3`

建议顺序：

```text
#92ADC0
#EB6E51
#9DBCBA
```

说明：

- 保留一个冷色骨架
- 以珊瑚橙形成清晰锚点
- 用灰绿补足第三组，避免整体过甜

### 6.2 `n = 4`

建议顺序：

```text
#92ADC0
#9DBCBA
#EB6E51
#F1B04A
```

说明：

- 先用两枚冷色建立秩序
- 再引入珊瑚橙和柔黄拉出暖色层次
- 这一组合适合常见 4 组科研图

### 6.3 `n = 5`

建议顺序：

```text
#92ADC0
#9DBCBA
#F1B04A
#EB6E51
#F3B1A6
```

说明：

- 保留完整 palette
- 浅粉放在最后作为最柔和的一组
- 若图中最后一组是弱化组、背景组或补充组，这一安排会更自然

### 6.4 当前使用建议

- 若对区分度要求更高，优先使用 `n = 3` 或 `n = 4`
- `n = 5` 时建议结合图例排序和数据分组语义使用
- 若未来支持 `n = 6`，需要额外补入一枚过渡色或重新调整策略
