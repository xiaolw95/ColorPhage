# ColorPhage 进阶 Family 设计启动草案

## 1. 文档目的

本文件用于启动 ColorPhage 的进阶 family 设计阶段。

当前 simple modes 已完成：

- `soft_breeze`
- `soft_harbor`
- `soft_mist`
- `vivid_core`
- `vivid_fresh`
- `vivid_drama`

并且真实 `ggplot2` gallery 已通过用户审阅。因此下一阶段重点应转向：

- 建立进阶 family 的设计流程
- 选择第一个 family
- 明确每个 family 的科研场景、审美边界、命名方式和展示标准

## 2. 进阶 family 的角色

简单模式解决的是：

- 用户不想做复杂选择
- 只需要 `soft` 或 `vivid`
- 快速获得高质量科研配色

进阶 family 解决的是：

- 用户希望图有更明确的风格气质
- 用户希望不同论文、不同研究主题有更强个性
- 用户希望 palette 名称更好记、有文化联想和品牌感
- 用户希望在保持科研审美的同时避免“期刊默认色板感”

## 3. 当前建议的 family 队列

建议第一轮进阶 family 仍保留此前规划的 6 类：

- `forest`
- `mineral`
- `ocean`
- `mist`
- `cinema`
- `retro`

但推荐设计顺序调整为：

1. `forest`
2. `mineral`
3. `ocean`
4. `retro`
5. `cinema`
6. `mist`

## 4. 推荐优先从 forest 开始

### 4.1 不建议第一个做 mist

`mist` 很重要，但它和当前 `soft` 系列较接近。

如果第一个进阶 family 就做 `mist`，用户可能不容易感知 simple mode 与 advanced mode 的差异。

### 4.2 不建议第一个做 cinema

`cinema` 有潜力做出强记忆点，但它和当前 `vivid_drama` 在功能上可能重叠。

如果太早做 `cinema`，容易把 advanced mode 变成 `vivid` 的加强版，而不是独立风格系统。

### 4.3 forest 的优势

`forest` 更适合作为第一个进阶 family，原因包括：

- 与 ColorPhage 的生命科学、生物生态、微生物语境天然贴合
- 和当前 simple modes 有足够区分
- 可以覆盖从温和到深色锚点的多种场景
- 很适合构造文化联想和记忆点
- 在科研图中可用于菌群、生态、分组比较、机制图、空间结构图

## 5. forest family 初步定义

### 5.1 核心气质

`forest` 应该是：

- 自然
- 稳健
- 有机
- 克制
- 有生命感但不幼稚

不应该是：

- 全绿同色系
- 户外品牌广告感
- 过深导致论文白底图发闷
- 过亮导致变成儿童插画或游戏 UI

### 5.2 科研适用场景

`forest` 适合优先展示在以下科研图中：

- 微生物群落组成图
- 菌株、宿主、处理组比较
- 生态梯度或环境分组
- 植物、土壤、肠道、环境微生物相关研究
- 空间转录组或组织微环境分类图
- 生物机制图中的模块化分组

### 5.3 配色结构建议

`forest` 不应只是绿色堆叠，而应采用：

- 森林绿作为稳重锚点
- 苔绿、鼠尾草绿、橄榄灰作为中间层
- 雾蓝、灰青作为冷色过渡
- 米白、浅土色作为缓冲
- 克制砖红、浆果色或赭橙作为少量强调

## 6. forest 首批候选分支

建议首批设计 3 个候选分支，而不是一次直接定稿：

### 6.1 `forest_canopy`

方向：

- 更稳重、更偏深林冠层
- 深绿、苔绿、灰青、米色、砖红锚点

科研实例：

- 微生物群落堆叠柱状图
- 不同生态位或样本来源的组成比较

记忆联想：

- 林冠、阴影、层次、生态位

### 6.2 `forest_moss`

方向：

- 更柔和、更偏苔藓和湿润地表
- 鼠尾草绿、苔绿、雾蓝、浅土色、柔和果实色

科研实例：

- 多处理组箱线图或散点图
- 肠道菌群、代谢组、环境因子关联

记忆联想：

- 苔藓、湿地、安静生长

### 6.3 `forest_ember`

方向：

- 更有叙事张力
- 深绿和冷灰绿为主体，加入受控暖色锚点

科研实例：

- 机制图、疾病组对照图、主结果强调图
- 需要突出干预组、疾病组或关键菌群的图

记忆联想：

- 森林里的余烬，低调但有视觉焦点

## 7. 关于文化联想命名的建议

用户希望命名轻松幽默，并能借用经典电影、游戏、歌曲或 IP 的联想。

当前建议采用“双层命名”：

- 代码层使用稳健、可维护、低法律风险的 ID
- 展示层和文档层使用更有记忆点的 nickname 或 cultural note

示例：

- `forest_canopy`：文档中可联想到“森林、冒险、生态位、林冠层”
- `forest_moss`：文档中可联想到“苔藓、雨后、安静生长”
- `forest_ember`：文档中可联想到“暗森林里的暖色火种”

不建议在第一版直接把受版权或商标保护的 IP 名放进函数参数或 palette ID。

原因：

- CRAN 或公开包文档中可能存在命名风险
- 用户调用时长期依赖这些 ID，后续改名成本高
- 更好的方式是把文化联想作为文案风格，而不是 API 稳定标识

## 8. forest 设计流程建议

建议沿用 simple modes 已验证的流程：

1. 风格讨论
2. 确定 2 到 3 个候选分支
3. 为每个分支构造 `n = 8` 初稿
4. 生成真实 `ggplot2` gallery
5. 用户审阅
6. 保留通过版本，淘汰不需要版本
7. 写入 R 包 palette 数据
8. 运行 `tools/check_package.ps1`
9. commit 并 push

## 9. 下一步

当前建议下一步：

> 先设计 `forest_canopy`、`forest_moss`、`forest_ember` 三个 `n = 8` 候选，并生成与 simple modes 相同格式的真实 `ggplot2` 预览页面。

这一步完成后，用户可以直观看到 `forest` 是否适合作为第一个进阶 family。
