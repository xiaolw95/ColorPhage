## 2. 当前规划目标

第一阶段 palette 库的目标不是追求数量，而是建立一个足够稳定、可扩展、能体现审美主张的起始颜色资产库。

优先服务以下需求：

- 简单模式可以快速输出高质量结果
- 进阶模式有足够清晰的风格差异
- `ggplot2` 接口可以尽快接入真实可用的 palette
- 后续 gallery 有足够代表性的展示内容

## 3. 来自参考图片的当前观察

基于 `pic` 文件夹中的抽样参考图，当前可提炼出以下偏好趋势：

### 3.1 整体偏好不是“极端艳丽”，而是“受控的高级感”

- 参考图多数不是高纯度全开
- 通常以低到中等饱和度作为主基调
- 常配合 1 到 2 个更鲜明的强调色

### 3.2 偏好多为“冷静底色 + 暖色锚点”或“柔和中间色 + 深色压轴”

常见组合逻辑包括：

- 蓝灰、雾蓝、青绿作为中间主色
- 米白、浅灰、淡粉作为缓冲色
- 珊瑚红、绯红、砖红、深海军蓝作为视觉锚点

### 3.3 喜欢的不是单色系，而是“跨色系但有秩序”的组合

- 很多参考图并非纯同色系扩展
- 常见做法是冷暖结合
- 但不会出现过多高冲突色同时抢焦点

### 3.4 绿色和蓝绿色系是重要资产

从当前样本看，以下方向值得优先纳入：

- 雾青
- 湖蓝
- 蓝灰
- 森林绿
- 苔绿
- 海水青

### 3.5 红色更偏“控制后的高光色”，而不是全局主色

- 鲜红、珊瑚红、砖红常用作强调
- 更适合作为 `vivid` 或 `cinema` 中的锚点色
- 不建议大量高纯红长期主导整组科研 palette

## 4. 第一阶段 palette 结构建议

建议将第一阶段颜色资产拆为两类：

1. 简单模式 palette
2. family 模式 palette

## 5. 简单模式 palette 规划

### 5.1 `soft`

建议首批设计 3 组：

- `soft_breeze`
- `soft_harbor`
- `soft_mist`

三组建议方向：

- `soft_breeze`：清新青绿 + 豆沙粉 + 浅蓝柔黄
- `soft_harbor`：雾蓝灰 + 灰青 + 柔黄珊瑚
- `soft_mist`：淡紫 + 雾蓝 + 米白浅粉

设计目标：

- 让用户在不做更多选择时也能得到温和、精致、论文友好的配色
- 作为简单模式的稳定默认资产

### 5.2 `vivid`

建议首批设计 3 组：

- `vivid_01`
- `vivid_02`
- `vivid_03`

三组建议方向：

- `vivid_01`：深海军蓝 + 灰蓝 + 冷白 + 绯红
- `vivid_02`：深青绿 + 雾青 + 浅白蓝 + 桃橙
- `vivid_03`：墨紫蓝 + 银灰蓝 + 冷白 + 亮红

设计目标：

- 在科研图语境内提供更强区分度和视觉锚点
- 强调冲击力，但避免廉价撞色

## 6. family 模式 palette 规划

### 6.1 `mist`

建议首批 2 组：

- `mist_01`
- `mist_02`

方向：

- 低饱和雾色组合
- 适度引入浅青、雾蓝、奶白、柔灰

### 6.2 `mineral`

建议首批 2 组：

- `mineral_01`
- `mineral_02`

方向：

- 蓝灰、石板蓝、冷灰、矿石青
- 可辅以极少量冷红或铁锈色作为点睛

### 6.3 `forest`

建议首批 2 组：

- `forest_01`
- `forest_02`

方向：

- 深森林绿、苔绿、鼠尾草绿、米白
- 可加入非常克制的砖红或果实色作为强调

### 6.4 `ocean`

建议首批 2 组：

- `ocean_01`
- `ocean_02`

方向：

- 海水青、湖蓝、浅水蓝、雾白
- 更适合未来扩展连续色板

### 6.5 `cinema`

建议首批 2 组：

- `cinema_01`
- `cinema_02`

方向：

- 深色底 + 高识别锚点色
- 强调戏剧张力和主次结构

### 6.6 `retro`

建议首批 2 组：

- `retro_01`
- `retro_02`

方向：

- 旧纸张米色、温暖土色、胶片绿、柔和砖红
- 保持温暖与秩序，不做脏旧感

## 7. 第一阶段数量汇总

当前建议的首批 palette 数量为：

- 简单模式：6 组
- family 模式：12 组

合计：

- 18 组首批 palette

## 8. 颜色数量覆盖建议

首批每组 palette 优先覆盖：

- `n = 3`
- `n = 4`
- `n = 5`
- `n = 6`

后续再扩展：

- `n = 7`
- `n = 8`

原因：

- 科研图常见分组数量集中在 3 到 6
- 先保证核心区间质量，比一开始追求大范围覆盖更重要

### 8.1 大数量类别的后续要求

虽然第一阶段优先覆盖 `n = 3` 到 `n = 6`，但包的长期目标不能停留在少数组别场景。

后续必须考虑支持：

- `n = 7`
- `n = 8`
- `n = 9`
- `n = 10`
- `n = 12`
- 甚至更高类别数场景

典型使用场景包括：

- 菌群组成或分类学分组图
- 单细胞聚类或细胞类型注释图
- 多疾病亚型比较图
- 多时间点、多处理组合图

因此需要明确：

- 第一阶段的 3 到 6 色方案是“核心种子”
- 第二阶段需要设计“高类别数扩展策略”
- 高类别数时不能简单线性插值或机械补色
- 必须重新考虑分层、分簇、明度节奏和相邻颜色区分问题

### 8.2 大数量类别的设计原则

当类别数扩展到 8 到 12 或更高时，建议遵循：

- 优先保证相邻颜色不混淆
- 不要求所有颜色都同等强烈
- 允许“主色群 + 辅色群”的层次化设计
- 必要时通过冷暖分组、明暗分组建立结构
- 不以“每个颜色都非常惊艳”为目标，而以“整组可用”为目标

## 9. 第一阶段设计方法建议

建议采用“参考图反推 + 内部规则筛选”的方式，而不是直接凭空生成。

推荐步骤：

1. 从参考图中归纳颜色组合规律
2. 先手动提出 palette 候选
3. 检查是否符合所属模式或 family 的气质
4. 在典型科研图中测试其表现
5. 通过后再纳入首批 palette 库

## 10. 当前下一步建议

当前最适合继续推进的事项：

1. 确认首批 palette 数量结构是否定稿
2. 开始逐组为 `soft` 和 `vivid` 设计候选 palette
3. 同步记录参考图来源和审美依据
